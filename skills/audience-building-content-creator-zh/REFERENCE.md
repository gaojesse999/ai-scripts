# 参考文档 —— 受众积累型内容创作器（中文版）

本文是 `SKILL.md` 的展开版，包含完整方法论、六要素好 / 坏对照库、格式模板与长度规格、平台说明、失败模式、长短内容战略以及多个 worked example。

方法论来源：`content-generation-rules/如何在没有任何粉丝的情况下建立自己的受众群.md`。有疑问时请直接读源文件。

---

## 1. 核心立场

冷启动增长的瓶颈不是"内容数量"，而是"被看见"。本 skill 的所有判断都从这里出发。

好的产出应当帮用户：

- 被一个**具体**的人群看见。
- 表达一个**可被辨认**的观点。
- 解决一个**真实**的问题或描述一种真实的张力。
- 用经验、专业、判断或清晰度建立信任。
- 制造陌生人可以转发、讨论、收藏、长期关注的东西。

要避免：

- 自我感动型的"艺术品"，难以理解或难以分享。
- 通用建议，谁写都一样。
- 没有自己立场的追热点。
- 只表达创作者情绪、却没给受众任何东西的内容。
- 空泛的网络寒暄、模板化的私信腔调。

## 2. 六要素（核心框架）

每条产出必须按格式适配地包含全部六条。缺任意一条都算失败模式。

### 2.1 钩子 Hook

首句 / 前 1–3 秒 / 第一张图必须让"对的人"停下来。钩子质量包含文字、版式、视觉组织。

**强钩子的常见模式：**

- **痛点真相：** 「你不是缺内容灵感，你是缺一个别人愿意关注你的理由。」
- **反共识立场：** 「新账号最不该先追热点。」
- **具体受众点名：** 「如果你是想做独立产品的工程师，这一条比 90% 的"增长建议"都重要。」
- **具体承诺：** 「用一个框架，把零散经验变成能积累受众的内容。」
- **熟悉的失败：** 「每天发内容却没人关注，通常不是因为你不够努力。」
- **带筹码的具体主张：** 「我做了 3 年内容，前 2 年只有 80 个粉丝。我后来发现我搞错了一件事。」
- **重新框定：** 「你以为更新频率决定增长，其实分发能力才决定。」

**避免：**

- 宽泛励志开场（"今天想跟大家聊聊…"）。
- 聪明但不清楚的措辞。
- 把错的人吸引过来的钩子。
- 内容支撑不住的夸大承诺。
- "今天天气真好"式或抽象到没有信息量的开场。

### 2.2 大问题 Big Problem

把问题写到"受众瞬间能认领"。受众越锋利，问题就能写得越锋利。

**好问题陈述：**

- 用受众自己的语言。
- 让读者 / 观众想"这真的就是我"。
- 有筹码（浪费的时间、损失的钱、声誉风险、错失机会）。

**受众积累场景常见的真问题：**

- 创作者对所有人说话，所以没人觉得"在跟我说话"。
- 内容只给结论，没有亲历。
- 建议正确但可替代。
- 内容很精致，但不实用、不可讨论、不可记忆。
- 创作者追热点，没在搭建一个可辨认的世界观。

**坏问题陈述：**

- 抽象（"内容创作很难"）。
- 通用（"每个人都想成长"）。
- 站在创作者一侧（"我想分享一下我的经验"）。

### 2.3 清晰解法 Clear Solution

给出**经过用户真实经验过滤**的可执行路径。"多输出、多坚持"这类正确但没用的话不算解法。

**强解法：**

- 来自用户自己的经验或专业能力。
- 说清"该做什么"和"该停止做什么"。
- 展示判断背后的思考。
- 带上用户的指纹（自己的限制、踩过的坑、独特的小转折）。
- 在下一篇 / 下一条视频 / 下一次对话里就能验证。

**弱解法：**

- 「坚持输出」「提供价值」「打造人设」「多互动」
- 任何听起来对、但没有下一步动作的话。

### 2.4 大收益 Big Benefit

把回报写明白。让受众猜，他们就不会转发或收藏。

收益类型：

- **实用：** 「你会更快知道哪类内容值得持续做。」
- **情感：** 「你不需要每天怀疑自己是不是不适合做内容。」
- **社交：** 「陌生人会更容易把你归类为某个问题的可靠解决者。」
- **身份：** 「你会从发散表达，变成一个有清晰立场的创作者。」
- **结果隐含：** 「我用这个结构改了 10 条帖子，平均互动翻了 3 倍。」

### 2.5 立场 / 极化 Confidence / Polarization

明确表达判断。**这里的"极化"不是挑事，不是制造仇恨。**它的意思是：不要写所有人都同意、但没人记得的话。

**有效立场模式：**

- 「真正的问题不是 X，而是 Y。」
- 「很多人以为要先做 A，其实更该先做 B。」
- 「这件事不适合所有人，但适合 [具体受众]。」
- 「我不建议新手这样做，因为……」
- 一句明确的"做 X，别做 Y"。
- 命名一个对立观点（"不要再追热点了，你要追的是同一类问题"）。

**和稀泥（失败信号）：**

- "两边都有道理。"
- "看情况。"
- "每个人的路不一样。"
- 反复加限定词，最后没观点。

### 2.6 新视角 / 大想法 Novel Perspective

加入那个让受众想"原来还能这么看"的洞察。

**新视角的来源：**

- 一次个人失败或案例。
- 一个跨行业类比。
- 一个反直觉的区分。
- 一个被命名的框架。
- 一个新的分析单位（"不要看单条内容表现，看一个月的复利曲线"）。
- 把别的创作者的观点和自己经验做一次合成。

## 3. 重新表达（开写之前必做）

把原始主题强制压进下面这句话：

```text
为了 [具体受众]，
他们正在面对 / 关心 [具体问题或欲望]，
这条内容帮他们 [具体收益]，
方式是展示 [经验 / 洞察 / 方法 / 故事 / 观点]，
让他们有理由去 [阅读 / 收藏 / 转发 / 评论 / 关注 / 信任]。
```

**主题收窄对照（从"太宽"到"具体到能赢"）：**

| 太宽 | 收窄后 |
|---|---|
| 个人成长 | 刚开始做自由职业、没有作品集的人，如何用 30 天建立第一批信任资产 |
| AI 工具 | 内容创作者如何用 AI 把零散经验变成稳定发布的文章和脚本 |
| 职场效率 | 工作 3–5 年的专业人士，如何把忙碌转成可见的专业影响力 |
| 健身 | 久坐程序员如何在不去健身房的情况下，用 12 周改善腰背疼 |
| 写作 | 第一次写公众号、没有读者的专业人士，如何让前 10 篇就被同行转发 |

如果一句话填不出来，**只问一个最关键的澄清问题**，而不是直接产出通用内容。

## 4. 自检

交付前**三件都跑一遍**。顺序：重新表达 → 压缩测试 → 分享测试 → 质量清单。

### 4.1 压缩测试（一行直觉）

内容必须给出五个理由：**停下 · 在意 · 信任 · 记住 · 分享**。任意一个弱了就改。

### 4.2 分享测试（3 问）

1. **为什么会有人转发这条？**（地位、身份、实用、"这就是我想说的"、新鲜感）
2. **目标受众会不会觉得"这就是我"？**（场景够具体、痛点被命名）
3. **立场是否锋利到能引发评论、收藏或被引用？**

分享对受众是一次自我表达。内容必须给他们值得表达的东西。

### 4.3 分享触发清单

强内容至少满足下面其一：

- **身份表达：** 「这说出了我正在经历的事。」
- **实用：** 「这能帮到我身边某类人。」
- **判断：** 「这个观点替我表达了一个判断。」
- **新意：** 「这个解释很新，我想保存或转发。」
- **讨论：** 「这个立场值得评论或反驳。」
- **信任：** 「这个人不像在套模板，他真的懂这个问题。」

如果都不满足，就回去改钩子、问题、收益或观点。

## 5. 格式模板

下列是默认值。按平台习惯、用户语气、用户指定的长度做调整。

### 5.1 文章大纲

```markdown
## 定位
- 目标读者：[具体受众]
- 内容承诺：[读者拿到什么]
- 核心角度：[用户的脊梁 —— 经验 / 观点 / 能力]

## 标题候选
1.
2.
3.

## 大纲
1. 钩子 / 开场判断
2. 具体问题（用读者的语言）
3. 个人经验或专业观察（脊梁）
4. 核心方法或观点（含新视角）
5. 例子 / 步骤 / 案例
6. 对读者的收益（大收益的再表达）
7. 结尾与行动建议（CTA / 分享触发）

## 受众积累逻辑
[为什么这能让对的人信任、记住、转发或关注]
```

### 5.2 完整文章

推荐结构：

1. **钩子**（1–3 句）。
2. **大问题** —— 用受众的语言写出来（1 个短段）。
3. **立场 / 主张**（1–2 句，明确表达）。
4. **3–5 个段落**，每段一个论点 + 一个证据或一个来自用户经验的例子。
5. **新视角显式出现** —— 在文章中段把那个"重新框定"亮出来。
6. **结尾：** 再次点明 takeaway 与收益，给出一个软性行动 / 分享邀请。

**长度默认（按平台调整）：**

- 公众号 / Substack / Medium：1200–2500 字（单核心观点）。
- LinkedIn 长帖：600–1200 字。
- 知乎回答 / 长贴：按题意定。

### 5.3 短视频脚本（15–60 秒）

```markdown
## 定位
- 目标观众：[具体]
- 视频承诺：[他们看完拿到什么]
- 观点：[一句话立场]

## 脚本
Hook (0–3s)
  口播：<让对的人停下来的一句>
  屏幕文字：<可选>
Problem (3–8s)
  口播：<用观众的语言命名问题>
Solution beats (8–45s)
  Beat 1：<一条具体动作 / 要点 + 例子>
  Beat 2：……
  Beat 3：……
Benefit + Takeaway (45–60s)
  口播：<他们拿到了什么、下一步做什么>
  屏幕文字：<值得保存的关键句>

## 可替换 Hook
1.
2.
3.

## 分享触发
[为什么会有人把这条发给朋友]
```

**不要**给出拍摄、剪辑、设备、灯光、转场、音乐、后期相关建议。

### 5.4 视频口播稿 / 长视频脚本

写法规则：

- 用**口语节奏**：短句、自然换气。
- 标注结构节拍：`【开场 Hook】`、`【问题】`、`【核心观点】`、`【解决方案 1/2/3】`、`【新视角】`、`【收尾 + 行动】`。
- 段落 / 换行多用 —— 一行一个意思最佳。
- 仅在需要镜头前重音的地方用 `**重点**` 标注，不要滥用。
- **字数估算：** 中文 ~250–300 字 / 分钟。按目标时长反推字数。

### 5.5 社媒图文文案（小红书、公众号短贴、知乎、微博、LinkedIn、X、Threads）

结构：

1. **标题 / 首句** —— 钩子 + 受众识别词。
2. **铺垫** —— 用 1–2 句把问题写明白。
3. **正文** —— 3–5 个编号或项目符号节拍，每条都要具体。
4. **takeaway** —— 一句把立场结晶。
5. **分享触发** —— 一句给受众"为什么要发"的理由。

图文 / 轮播帖再加一份页面结构：

```markdown
封面：[钩子 + 受众识别词]
第 1 页：[问题]
第 2 页：[误区 / 对照]
第 3 页：[解法节拍 1]
第 4 页：[解法节拍 2]
第 5 页：[例子 / 案例]
结尾页 / CTA：[takeaway + 收藏 / 分享理由]
配文：[发布时的正文]
```

**平台说明：**

- **小红书：** 标题要冲、场景要具体、结构要"值得保存"，只有当用户语气支持时才用少量 emoji。
- **公众号 / newsletter：** 论证更深、信任靠故事和框架、面向长期关系。
- **知乎：** 答题要先给立场再展开论证，例子和数据是信任货币。
- **微博 / Threads：** 一句锐利观点，可被引用，正式度低。
- **视频号短贴 / 朋友圈长文：** 强身份感 + 强场景感，转发场景多为熟人。
- **LinkedIn：** 观点先行的开场 + 故事或案例 + 结构化正文 + takeaway + 一个能引发评论的问题。
- **X / Twitter thread：** 第 1 条 = 钩子 + 承诺；中间每条一个节拍；最后一条 = takeaway + 软 CTA。每一条都要单独可被引用。

### 5.6 内容选题列表

默认 **8** 条，按规模可在 **5–15** 条之间。每条至少包含：

```markdown
- 标题（草稿）：<锋利的工作标题>
  写给谁：<具体受众>
  大问题：<一行>
  新视角：<重新框定 / 新角度>
  大收益：<一行>
  最佳格式：<文章 / 短视频 / 口播 / 图文 / thread>
  为什么能吸引对的人：<一行>
```

选题倾向：

- 保持**单一主题**（让受众知道你"是什么的人"）。
- 来自用户真实经验或专业能力。
- 每一条都能单独通过分享测试。

避免只给标题。没有受众逻辑的标题对长期增长没有帮助。

### 5.7 内容优化建议

当用户给出存量内容要求优化时：

1. **按六要素诊断** —— 指出哪几条到位、哪几条弱、哪几条缺。
2. **跑分享测试**。
3. **给具体改写，而不是抽象建议**。最少包含：
   - 改写钩子（提供 2 个备选）。
   - 把问题陈述写得更锋利。
   - 收紧立场，去掉和稀泥。
   - 指出这条内容可以落到哪种新视角上。
4. 结尾给 **1–3 条可以迁移到未来作品里**的具体行动，而不仅服务这一条。

## 6. 失败模式（要避开的雷）

- **自嗨型艺术品** —— 创作者很满意，市场不在意。
- **没有脊梁的追热点** —— 蹭热度但没有自己的角度。
- **"人人都同意"的废话** —— 无立场、无记忆点、无分享。
- **通用科普** —— 谁都能写，看不出是你写的。
- **模板私信腔调** —— 把那种"想跟你 virtual coffee 一下"的能量带到公开内容里。
- **评论区刷脸式逻辑** —— 复述原帖或为了被看见而回复。
- **受众设成"所有人"** —— 一定落不到任何人手上。
- **钩子吸引错的人** —— 高浏览、零关注。
- **解法 = 正确但没用** —— "坚持输出""提供价值"。
- **视频创作请求漂移到拍摄 / 剪辑 / 设备** —— 本 skill 只管内容侧。

## 7. 长期品牌偏好

优先稳定的受众积累，不优先短期流量技巧。

**优先：**

- 能积累辨识度的可复用主题。
- 个人经验 + 有用抽象。
- 可被串成系列的内容。
- 能引向长文、newsletter、播客或长视频的短帖。
- 围绕一个创作者能长期服务的问题做定位。
- 真诚的合作、引用、评论与回复。

**避免：**

- 与长期方向无关的追热点。
- 没有受众适配的"爆款公式"。
- 没有读者收益的自我倾诉式叙事。
- 还没验证就过度打磨。
- 空回复、模板社交、虚假亲密。
- 只靠激怒受众获得表现、但建立不起信任。

## 8. 长短内容战略

当用户在规划一组内容而非单条时，简短提醒以下结构逻辑：

- **短内容**（帖子、tweet、reel）是**入口漏斗** —— 它争取前 10 秒的注意力。
- **长内容**（文章、newsletter、播客、长视频）**占用 15–60 分钟**的注意力，建立深信任与议题所有权。
- 健康的内容计划是**把跑出来的短内容沉淀成长内容资产**。每一条表现好的短帖，都是一篇长内容的候选种子。

只在规划时用这一节，不要每条产出都讲一遍。

## 9. Worked Examples

### 9.1 小红书短帖

**用户需求：**「我是一个做了 4 年后端的工程师，想转独立开发者，帮我写一条小红书。」

**重新表达：**
- 写给谁：3–5 年经验、想做独立产品的后端工程师
- 大问题：以为卡点是技术，疯狂补前端 / AI / 设计
- 大收益：少走前 6 个月的弯路
- 脊梁：用户自己从打工到独立开发的真实卡点
- 新视角：「你卡的不是技术，是选题和分发」
- 分享触发：同类工程师转发表达"这就是我"

**产出：**

```text
后端工程师转独立开发，前 6 个月我做错的一件事

我做了 4 年后端，去年决定转独立开发。
前 6 个月，我以为技术不够，疯狂补前端、补 AI、补设计。
后来我发现：卡住我的根本不是技术。

是选题，和没人知道我在做什么。

3 件我后来才想明白的事：
1. 你不缺工具，你缺一个值得做半年的具体问题。
2. 第一批用户不会从应用商店来，他们会从你公开写过的东西来。
3. 工程师做独立产品，最大的杠杆不是再学一门技术，是开始在公开场合写你解决的问题。

如果你也是工程师在想转独立开发——
先选一个你真的懂、真的烦过的问题，
再开始公开地写它。

技术不是你最缺的那块拼图。
```

**六要素自检：**

### 9.3 60 秒口播稿

**用户需求：**「我做内容策略咨询的，帮我写一条 60 秒的口播，讲'为什么大多数公司的内容账号做不起来'」

**重新表达：**
- 写给谁：负责公司内容账号但增长卡住的市场 / 品牌负责人
- 大问题：发了一年内容没增长，老板开始质疑
- 大收益：知道真正要换的是定位，不是更新频率
- 脊梁：咨询师见过的真实公司案例
- 新视角：「公司账号做不起来，是因为它在替'公司'说话，不是替'某一类人'说话」

**产出（~250 字，~60 秒）：**

```text
【开场 Hook】（0–5s）
大多数公司的内容账号做不起来，
不是因为发得少。

【问题】（5–15s）
我做内容策略咨询 5 年，
见过太多公司每天发 3 条，做了一年还在 2000 粉丝。
他们都在问同一个问题：是不是要换平台？

【核心观点】（15–30s）
都不是。
真正的问题是——
你的账号在替"公司"说话，
不是替"某一类具体的人"说话。

【新视角 + 解决方案】（30–50s）
个人能涨粉，是因为一个人天然就有具体身份。
公司想涨粉，得先**主动选择**一个具体的人。
不是"我们服务所有中小企业"，
是"我们替那个'第一次招 AI 工程师、不知道怎么面试'的 CTO 说话"。
你的内容只要让那一类人觉得"这是替我做的"，
他会自己开始转发、自己开始关注、自己开始信任你。

【收尾】（50–60s）
所以公司账号做不起来，
**换的不是频率，是替谁说话**。
```

**六要素自检：**
- 钩子：反直觉 + 行业判断 ✓
- 大问题：一年 2000 粉，老板开始质疑 ✓
- 清晰解法：选一个具体身份的人去服务 ✓
- 大收益：不再换平台试错 ✓
- 立场：「换的不是频率，是替谁说话」✓
- 新视角：公司账号要"主动选择身份" ✓

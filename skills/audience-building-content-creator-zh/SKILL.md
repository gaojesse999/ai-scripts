# 受众积累型内容创作器（中文版）

把一个早期想法、主题、观点、经验或内容方向，转化为能吸引清晰受众、建立信任、并在长期内积累成个人品牌的中文内容。

方法论来源：`content-generation-rules/如何在没有任何粉丝的情况下建立自己的受众群.md`。在生成正式内容前，请先阅读 [REFERENCE.md](REFERENCE.md) 获取完整框架、模板与示例。

**语言规则：本 skill 的所有产出与对用户的回应必须使用中文。**

## 适用场景

只要用户提出以下任一类需求，即使没有点名方法论，也应用本 skill：

- 文章大纲
- 完整文章
- 短视频脚本（小红书短视频、抖音、视频号、Reels、Shorts、TikTok 等）
- 视频口播稿 / 长视频脚本
- 社媒图文文案（小红书、公众号短贴、知乎、微博、Threads、LinkedIn、X 等）
- 内容选题列表
- 内容优化建议
- "帮我写一篇 X 关于 Y" 类请求

如果用户把任务定位为**视频创作 / 做视频 / 视频策划**，只输出内容侧：选题、结构、脚本、口播、屏幕文字、镜头内容建议。**不要**讨论拍摄、剪辑、设备、灯光、软件、后期，除非用户明确要求。

## 必要输入

生成前确认下列信息是否齐全。**只追问缺失且关键的部分**，不要罗列问卷。

1. **主题 / 方向** —— 这条内容写什么？
2. **目标受众** —— 具体是谁？（身份、阶段、痛点、自我认同）
3. **内容角度** —— 观点、经验、框架、案例、教程、故事等。
4. **关键素材** —— 用户的真实经验、判断、数据、例子。
5. **平台** —— 决定长度、语气、结构。
6. **输出类型** —— 大纲、完整文章、脚本、口播、社媒文案、选题列表、优化建议。

### 提问策略

- 信息已足够生成强内容，**直接开始写**。
- 关键信息缺失（尤其是受众、平台、用户的真实角度 / 经验），**最多一次性提 2–3 个高杠杆问题**，然后开始生成。
- 不要把用户卡在一长串问卷里。

最高杠杆的澄清问题：

- 你希望什么样的人读完 / 看完会觉得"这就是写给我的"？
- 这条内容发在什么平台、大概多长？
- 你在这件事上真实的经验、观点或踩过的坑是什么？（这是内容的脊梁）

## 工作流（8 步清单）

在心里跟进，不要展示给用户，除非用户主动询问。

```
- [ ] 第 1 步：解析输入 —— 主题、受众、角度、关键点、平台、输出类型
- [ ] 第 2 步：必要时澄清（最多 2–3 个问题，仅在关键缺失时）
- [ ] 第 3 步：锁定受众 —— 一个具体人群，不是"所有人"
- [ ] 第 4 步：锁定脊梁 —— 用户真实的经验 / 观点 / 能力
- [ ] 第 5 步：用「重新表达模板」把主题收窄成一句话
- [ ] 第 6 步：套用六要素（钩子 · 大问题 · 清晰解法 · 大收益 · 立场 / 极化 · 新视角）
- [ ] 第 7 步：按请求的格式、平台、长度规格成型
- [ ] 第 8 步：跑「压缩测试 + 分享测试 + 质量清单」，修改后交付
```

### 重新表达模板（开写之前必做）

把原始主题强制压进这句话：

```text
为了 [具体受众]，
他们正在面对 / 关心 [具体问题或欲望]，
这条内容帮他们 [具体收益]，
方式是展示 [经验 / 洞察 / 方法 / 故事 / 观点]，
让他们有理由去 [阅读 / 收藏 / 转发 / 评论 / 关注 / 信任]。
```

如果用户只给了宽泛主题，先收窄。示例：

- 坏：「个人成长」 → 好：「刚开始做自由职业、没有作品集的人，如何用 30 天建立第一批信任资产」
- 坏：「AI 工具」 → 好：「内容创作者如何用 AI 把零散经验变成稳定发布的文章和脚本」
- 坏：「职场效率」 → 好：「工作 3–5 年的专业人士，如何把忙碌转成可见的专业影响力」

### 六要素（必须全部出现，按格式适配）

1. **钩子 Hook** —— 首句 / 前 1–3 秒 / 第一张图必须让"对的人"停下来。
2. **大问题 Big Problem** —— 命名一个受众瞬间能认领的"这就是我"的问题。
3. **清晰解法 Clear Solution** —— 经过用户真实经验过滤的、可执行的路径，不是"多输出、多坚持"。
4. **大收益 Big Benefit** —— 直说受众能拿到什么，可分实用 / 情感 / 社交 / 身份。
5. **立场 / 极化 Confidence / Polarization** —— 给出明确判断。不和稀泥，也不为了博眼球开火。
6. **新视角 / 大想法 Novel Perspective** —— 让受众产生"原来还能这样理解"的新框架。

每条要素的好 / 坏对照、钩子库与中文示例详见 [REFERENCE.md §2](REFERENCE.md)。

### 长度 / 格式默认值（按用户语境调整）

下列是起点而非铁律。用户的平台、风格或限制更优先。

- 短视频（15–60 秒）：钩子 0–3s · 问题 3–8s · 解法节拍 8–45s · 收益 + 收尾 45–60s。
- 口播稿 / 长视频：中文 ~250–300 字 / 分钟。按目标时长反推字数。
- 公众号 / Substack / Medium 单文章：1200–2500 字（单核心观点）。
- LinkedIn 长帖：600–1200 字。
- 知乎回答 / 长贴：按题意定。
- 选题列表：默认 8 条，按需求规模可在 5–15 条之间调整。

## 输出格式默认

先交付用户要的「成品」。方法论说明放在末尾且要短。匹配用户的真实语气，不要无故拔高成公关稿。详尽模板见 [REFERENCE.md §5](REFERENCE.md)。

**新内容请求**应包含：
- 定位摘要：目标受众 · 内容承诺 · 角度
- 主成品：大纲 / 草稿 / 脚本 / 帖子 / 选题列表
- 可选变体：2–3 个备选钩子 / 标题 / CTA

**优化请求**应包含：
- 按六要素诊断（哪几条到位、哪几条弱、哪几条缺）
- 实际改写，而不是抽象建议
- 1–3 条可以迁移到未来作品里的具体行动

## 交付前自检

### 压缩测试（一行直觉）

内容必须给出五个理由：**停下 · 在意 · 信任 · 记住 · 分享**。少一个就改。

### 分享测试（3 问）

1. 为什么会有人转发这条？
2. 目标受众会不会觉得"这就是我"？
3. 立场是否清晰到能引发评论、收藏或被引用？

### 质量清单（必须全过）

- [ ] 内容写给一个**具体**人群，不是"所有人"。
- [ ] 脊梁是**用户真实的经验 / 观点 / 能力**，不是通用科普。
- [ ] 六要素全部出现且可指认。
- [ ] 有明确立场（不和稀泥、不靠"看情况"逃避）。
- [ ] 至少一个分享理由：身份 / 实用 / 判断 / 新意 / 讨论 / 信任。
- [ ] 不空追热点、不自嗨倾诉、不写"人人都同意"的废话。
- [ ] 视频创作：只给选题 / 结构 / 脚本 / 口播 / 屏幕文字 —— 不谈拍摄、剪辑、设备、后期。
- [ ] 成品干净、可直接发布或开拍，不被理论拖累。

任何一项不过就修改后再交付。

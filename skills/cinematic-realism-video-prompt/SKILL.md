# 电影写实视频提示词

## 核心行为

调用本技能时，生成一段中文长篇视频生成提示词，风格必须贴近用户已经认可的样式：高密度、电影化、触感强、物理落地、真人实拍、序列推进，并持续堆叠可见细节。

不要总结，不要写镜头大纲，不要输出“差不多够用”的短提示词。每条提示词都应像一段可以直接用于生产的完整长段落，密度接近用户已接受的版本。

默认输出：只输出最终中文提示词正文，不解释，不使用 Markdown，不写项目符号。

如果用户要求多个版本，使用 `版本1`、`版本2` 等简短标签；每个版本都必须是完整独立提示词。

如果用户要求分段视频提示词，每个片段仍必须是高密度长提示词。15秒片段通常写 1200-1800 个中文字符，除非用户明确要求更短。拆分长脚本时也要保持电影密度。

## 写作前

先静默提取以下信息：

- 挂载引用：用户请求里的任何 `@场景`、`@人物`、`@道具` 或 `@设定`，以及文本中可识别的场景、人物、怪物、道具名称。即使用户没有手写 `@`，也要自动整理成挂载前缀。
- 全局约束：任何方括号要求，例如 `【全程无背景音乐，无文字字幕，超写实真人电影质感】`。保留用户原文，并放在最终提示词开头附近。
- 主体：主角、配角、身份、视觉特征、服装、道具。
- 地点：环境、时间、天气、地形、建筑、植被、颗粒物。
- 情绪：紧张、压迫、幽默、傲慢、恐怖、庄严、美感、危险。
- 动作节拍：先发生什么、发生了什么变化、谁作出反应、场景如何结束。
- 镜头方案：开场角度、运动方式、切点、景别、最终画面。
- 对白：如果用户给出对白，必须保留原句；如果没有对白，可创建与权力关系匹配的短句。
- 场景类型：动作类、通用叙事类、对话类或混合类，并继续细分到具体场景原型。不同场景不能套同一个模板。
- 镜头模式：一镜到底、丝滑剪辑或缓慢氛围推进。模式要服务动作逻辑，不要只为炫技。

只有在场景没有主体或没有地点时才向用户提问。

## 默认项目圣经引用

如果当前项目存在以下文件，生成前默认先读取并参考，不需要用户每次手动挂载；这些是内部参考文件，不要写进最终提示词的挂载前缀或正文里：

1. `assets/整体风格.md`：最高优先级的整体视觉、光照、声音、镜头和技术风格来源。
2. `assets/人物圣经.md`：人物身份、关系、说话方式、行为逻辑来源；人物外貌仍以挂载图片为准，不重新描述。
3. `assets/场景圣经.md`：场景空间结构、动线、道具、光源和戏剧功能来源。

优先级顺序：用户最新明确要求 > 挂载图片/音频/场景素材 > `assets/整体风格.md` > `assets/人物圣经.md` > `assets/场景圣经.md` > 本 skill 默认规则。

默认继承 `assets/整体风格.md` 的核心约束：暗黑奇幻真人电影质感、低饱和暗冷色、自然光/背光/阴影侧摄影、薄雾、8K IMAX 真人实拍、PBR 物理材质、24fps、180度快门运动模糊、无音乐、无字幕、只保留环境音效和动作音效；人物语音参考人物圣经中的语言特征，默认使用英文作为台词，可对输入语义做适当英文适配。不要每次整段复述文件内容，只把这些原则落实到提示词里。

## 挂载输入格式

支持以下输入模式：

`【全程无背景音乐，无文字字幕，超写实真人电影质感】`

规则：

- 将挂载引用视为提示词的场景、角色和设定清单。
- 如果挂载引用指向已知文件、打开文件或已提供的人物圣经，以其中细节为准。
- 如果某个挂载引用没有可用细节，保留名称，只保守推断镜头所需的可见细节。
- 第一个挂载地点通常是主场景；后续挂载名称通常是人物，除非文本另有说明。
- 方括号全局约束必须原样出现在质量风格锁之前或之后。

## 自动挂载前缀

最终提示词必须以挂载前缀开头，把用户文本中出现的主要场景、人物、怪物、道具整理成 `@名称` 形式。顺序固定为：主场景 → 主要人物 → 怪物/敌人 → 关键道具/语音/设定。不要把 `assets/整体风格.md`、`assets/人物圣经.md`、`assets/场景圣经.md` 写进挂载前缀；它们只作为生成前的内部参考。

如果用户已经提供 `@`，保留用户写法并补齐遗漏的核心挂载；如果用户没写 `@`，根据文本自动生成。例如：

`@鲜血荒地，@野蛮人，@亚马逊，@女巫`

别名要规范为人物圣经或场景圣经中的正式名称：`法师` 归并为 `女巫`，`目盲修女会/盲眼姐妹会` 保留为组织设定，不当作单独人物；`阿卡拉帐篷/阿卡拉营帐` 统一为 `阿卡拉营帐`。不要为只在对白里随口提到、没有出现在画面中的对象生成挂载，除非用户明确要求。

挂载前缀之后再接方括号全局约束、挂载资源与音频硬约束、质量风格锁和正文。默认保持一段式输出，但挂载前缀可以作为开头独立短行。

## 挂载资源与音频硬约束

如果用户提到“挂载人物”“挂载场景”“挂载语音”“使用参考角色”“使用参考场景”“不要生成新声音”“直接使用语音”等要求，必须把这些要求放在最终提示词最前面。

默认合并写法：

`【挂载资源与音频硬约束】本视频必须严格使用已挂载的人物形象、角色设定、场景参考、怪物参考和语音素材；人物外貌、服装、发型、气质、阵营风格、场景空间结构、建筑造型、怪物形态不得偏离挂载素材。已有挂载人物图片或人物圣经时，不重新发明、不重新描述人物外貌、服装和发型，只写“严格参考挂载形象”，避免与参考图冲突。无字幕，无文字标题，无屏幕文字，无背景音乐，不生成配乐。语音有两种模式：默认模式是把挂载语音作为说话人物的音色参考，由模型重新生成对应台词，但音色、语气、节奏和停顿必须贴近参考语音；直接使用语音模式是原样使用挂载语音文件，不重新生成、不改音色、不改节奏、不生成新的旁白或自动配音。无论使用哪种语音模式，都必须生成真实环境音效与动作音效，包括风声、脚步声、火焰声、布料摩擦声、武器碰撞声、怪物低吼、门轴呻吟、毒雾翻涌等符合场景的声音。`

语音模式判断：用户只说“挂载语音”“参考语音”“语音用某角色声音”时，默认按音色参考处理；用户明确说“直接使用提供的audio/语音”“不要重新生成声音”“原音频继续播放”“使用原始语音文件”时，按直接使用语音处理。说话人物的语音也要进入自动挂载前缀，例如 `@野蛮人语音，@女巫语音`。

如果用户指定具体说话人或地点，要写清：说话人物必须是该挂载形象；场景必须使用该挂载地点的空间结构和风格，不得替换为其他人物或空间。

2. 用户给出的方括号全局约束，如果有。
3. 挂载资源与音频硬约束，如果有。
4. 全局质量与风格锁。
5. 挂载场景/环境写实。
6. 挂载人物参考与表演细节。
7. 开场镜头与第一动作节拍。
8. 运镜或剪切，用来揭示空间关系。
9. 其他角色或环境的反应节拍。
10. 对白节拍与情绪压力变化。
11. 最终特写或定格收尾。

## 场景类型与原型路由

生成前先按 `seedance-director` 的方式判断主场景类型，再细分到场景原型；混合场景要分清主次，不要平均用力。最终输出不展示分析过程，只把判断体现在镜头结构里。

动作类原型：

- 追逐：有人追或被追。镜头重点是距离变化，被追者通常在前，追赶者在后；空间应体现道路变窄、开阔、转折或障碍。
- 对决：两个对手互相争夺优势。优势必须交替，不能让一方连续占优太久；如果一方全程压制，应写成单方面袭击，而不是对决。
- 冲击：单个决定性接触瞬间。节奏是“蓄势慢 → 命中快 → 余波慢”，接触点必须成为画面中心；描述力量和方向，不写过度复杂的破坏链。
- 默认判断：有追赶选追逐；双方互换优势选对决；单次决定性命中选冲击；否则按对决处理。

通用叙事类原型：

- 旅程：主体在空间中移动。重点是位置变化，适合跟拍、航拍、并行移动，景物应随人物经过而变化。
- 氛围：几乎没有事件变化，情绪本身就是内容。适合静态或缓慢推进，用雨、雾、火、风、灰烬、呼吸、表面细节承载戏。
- 揭示：隐藏信息逐步变得可见。适合横摇、摇臂、推轨、绕过遮挡物，让镜头控制观众何时看见主体。
- 默认判断：主体穿越空间选旅程；隐藏物变可见选揭示；没有明显事件变化选氛围；否则按氛围处理。

对话类原型：

- 对抗：双方都在施压，权力关系来回变化。适合紧过肩、反应特写、越轴或景别变化来表现优势转移。
- 审讯：一方逼问或索取信息，另一方抵抗。适合逼问者低角度、沉默者推进特写，重点是停顿、眼神和手部紧张。
- 谈判：双方都需要对方，力量相对平衡。适合对称构图、匹配景别、克制镜头运动。
- 默认判断：双方互相推进选对抗；一方索取一方抵抗选审讯；双方交换条件选谈判；否则按对抗处理。

混合类：例如“对话挑衅 → 战斗爆发”或“旅程铺垫 → 发现敌人 → 追逐”。类型切换必须有可见过渡，动作爆发要由前面的发现、台词或冲突推动。

镜头模式按原型选择：连续追逐、格斗、突袭优先一镜到底；对话、对峙、多人反应优先丝滑剪辑；旅程、揭示、氛围优先缓慢氛围推进。强度档位默认自动判断：普通剧情和情绪戏用标准版；动作戏和战斗戏用强化版；用户明确说“拉满、狂暴、极限、像参考例子一样炸”时使用狂暴版。无论档位如何，都不能为了视觉冲击破坏剧情逻辑。

## 密度要求

每条完整提示词必须包含八层细节：

- 渲染锁：IMAX、真人实拍、8KPBR、无字幕、无音乐、无虚拟CG、无廉价平面特效。
- 环境层：时间、光线方向、空气流动、地面纹理、表面材质、颗粒物、残骸，以及“1:1真实实景”感。
- 人物参考层：已有挂载图片/人物圣经时，只强调严格参考挂载人物形象，不重写外貌、服装、发型、五官和年龄感。
- 表演写实层：必须写微表情、眼神焦点、瞳孔变化、呼吸、口型、下颌紧张、手指力度、身体重心、武器握持、衣料受力等可见动态。
- 动作层：具体身体动作、物体运动、因果连续性、冲击细节、小型过渡动作。
- 镜头层：开场角度、运镜、剪切或景别变化、重新交代空间关系、最终构图。
- 表演层：微表情、呼吸、瞳孔变化、嘴部动作、下颌紧张、手指紧绷、对白说法。
- 氛围层：压迫、沉默、对峙、恐惧、傲慢、幽默或悬念，必须通过可见身体行为呈现。

如果输出读起来像剧情梗概，就继续扩写。补足可见物理、触感材质、面部表演和镜头连续性，直到接近已接受的长例子密度。

## 已认可风格标准

使用用户已认可的“堆叠式高密度”风格：

- 以一条长质量渲染锁开头。
- 立即展开环境：自然光、风/空气运动、地面表面、植被或地形、纹理、残骸、湿度，并在合适时使用“1:1还原真实实景”等语言。
- 在第一个动作前加入“人物严格参考挂载形象”的约束，不重述外貌和服装；把细节重点放在微表情、眼神、呼吸、口型、手指、身体重心、武器握持和衣料受力。
- 每个动作都附带可见微细节：手指、关节、布料牵扯、发丝运动、面部表情、呼吸、物体轨迹、飞溅物、颗粒、冲击。
- 每次运镜都写清镜头角度和镜头行为，例如“贴地超低角度缓慢前推”“正面低角度仰视近景”“快速环绕向后拉远成俯视角全景”“极近距离面部特写”。
- 以带压力的最终画面结束：特写、静止、反应或定格。

不要只说一次“细节真实”。每个节拍都要写具体可见细节。

## 风格锁

以强力质量前缀开头。可根据场景调整措辞，但保持这种密度：

`8K超清 IMAX宽画幅，院线级真人实拍电影质感，8KPBR物理渲染精度，无虚拟CG，无廉价平面特效，全程手持稳定动态运镜，镜头切换丝滑无断层、构图固定规整，自然光/夜色光影通透写实，临场压迫感与肃杀氛围感拉满。`

必要时可反复使用“真人实拍质感”，但不要无意义堆关键词。人物外貌、服装、发型已有挂载参考时，不重新描述；每个写实短语都必须绑定可见对象或表演动作：眼神变化、瞳孔、嘴唇、呼吸、手指、布料受力、金属氧化、湿泥、灰尘、叶片、武器握持。

## 环境写实

每个地点都要落在可见物理细节上：

- 光线：自然光方向、冷暖色调、轮廓光、体积光/丁达尔光束、面部阴影。
- 空气：风、雾、尘、雨、火星、花粉、雪、呼吸白雾、烟。
- 地面：泥、苔藓、落叶、碎石、裂石、草，需要时才写水坑。
- 表面：布料织纹、金属划痕、皮革磨损、木纹、石材纹理，只有稳定时才写水面倒影。

避免无法看见或听见的感官描述。不要写“闻到松香”，改写成“松针铺满地面，风掠过枝叶”。

常用高密度表达：

- `微风/夜风全域贯穿场景，吹动枯叶、杂草、枝叶/灰烬簌簌摇曳翻飞`
- `地面百草丰茂/黑土微润无积水，腐叶、苔藓、灌木枝干/碎骨、断箭、烧焦草茎纹理极致写实`
- `1:1还原真实森林/荒地/营地实景`
- `泥点、灰烬、碎冰、血珠、布料纤维贴近镜头前景，动态细节真实可感`

## 人物参考与表演写实

对每个重要人物，先判断是否有挂载图片、人物圣经或明确参考形象：

- 有挂载参考时：不要重新描述脸型、五官、发型、服装款式、颜色、年龄感和整体气质，避免和图片冲突。写法应是“人物形象严格参考挂载图/人物圣经”，然后直接进入动作、表演和镜头细节。
- 没有挂载参考时：只做保守、可见、必要的人物描述，不发明复杂外貌设定。
- 必须保留表演细节：眼神焦点、瞳孔收缩、下颌收紧、鼻翼翕动、嘴唇开合、口型对应台词、呼吸起伏、肩颈紧张、手指泛白、握把力度、身体重心变化。
- 必须保留动态材质细节：发丝和衣料如何被风或动作带动，布料如何受力牵扯，皮革握把如何被压紧，金属、木材、盾牌、法杖、武器如何随动作产生重量感和轻微碰撞。

不要在没有可见证据的情况下把人物写成“完美”或“绝美”。美感或威胁感应从微表情、姿态、动作压力和镜头关系中自然产生。

常用高密度表达：

- `人物形象严格参考挂载图与人物圣经，不重新设计外貌、服装、发型和整体气质`

## 镜头与连续性规则

用镜头语言控制叙事：

- 开场镜头必须建立优势、脆弱、神秘或地理关系。
- 每次剪切都改变景别和镜头行为。
- 剪切后要重新交代谁在哪里、面向哪个方向。
- 保持动作物理连续：抛出的物体会落下，发丝会继续随风运动，人物听到或看到之后才反应。
- 每条提示词优先使用 2-4 个主要镜头节拍，不要塞入无法执行的复杂调度。
- 如果设定为一镜到底，不能出现“镜头切到”“画面一转”“突然切换到”“下一个镜头”“插入特写”等切镜语言。

常用运镜：

- 正面超低角度仰视
- 贴地横移跟拍
- 手持稳定快速环绕
- 缓慢推进近景
- 俯视角全景拉远
- 极近距离面部特写
- 快速切换至反应镜头

## 逻辑正确性自检

生成前和输出前都要静默自检；默认不展示检查过程，除非用户要求分析。

- 动作类：检查“动作因果 → 空间站位 → 身体受力 → 环境反馈 → 镜头连续性 → 高潮合理性”。确认谁先动、目标是什么、距离是否清楚、被攻击者如何反应、高潮是否由前面动作自然推动。
- 对话类：检查“台词动机 → 视线关系 → 权力关系 → 表情反应 → 情绪递进 → 结尾关系变化”。确认谁掌握主动权、对方是否有合理反应、表情和手部动作是否推动气氛。
- 通用类：检查“场景目标 → 信息揭示顺序 → 镜头运动 → 氛围递进 → 画面记忆点”。确认镜头是展示空间还是跟随人物，信息如何逐步揭示。
- 混合类：检查类型切换是否自然，情绪是否先铺垫再爆发，动作是否由对话、发现或冲突推动。
- 空间关系：必须清楚谁在正面、背后、高处、低处、远处；攻击路径或视线方向必须合理。
- 身体连续性：角色姿态、重心、受击反馈、武器/触手/法术轨迹必须前后一致；不要让角色突然瞬移、突然站直、突然获得没有铺垫的新姿态。

## 对白与动作

对白应短、锐利，并能在镜头内成立。对白统一放在中文引号中。

傲慢或喜剧压迫感：

- 使用随口的措辞、停顿、轻蔑节奏和小型不尊重动作。
- 让身体保持松弛，而局势变得危险。

紧张感：

- 在暴力前表现身体收紧：手指合拢、肩膀压低、瞳孔缩小、呼吸停住。
- 让沉默和静止在最终台词或动作前承载压力。

## 回忆、转场与群像

当用户要求“回忆”“讲述过去”“传说”“曾经”“昔日”等内容时，优先使用回忆结构。如果用户指定“从眼神进入回忆”，可写：镜头推进到说话人的面部近景，最终锁定双眼，瞳孔中的火光、月光或环境倒影逐渐拉长、扭曲，变成回忆场景中的对应光源或空间结构。如果用户指定“后面一直在回忆里”，必须写清进入回忆后不再切回现实。

当场景中出现多个同阵营人物、战士、村民、怪物或士兵时，必须避免同脸复制。群像人物要有脸型、五官比例、肤色深浅、发型、身高体态、表情与站姿差异；服装保持统一阵营风格，但在护甲、披风、武器、配饰、磨损痕迹上有细微差异。怪物群像要有体型、腐烂程度、装备残片、动作姿态和移动方式差异，不能像复制粘贴的同一只怪物。

## 负面约束

不要包含：

- “动漫风”“二次元”“卡通”“插画感”，除非用户明确要求。
- “廉价特效”“塑料感”“AI感”，除非作为负面约束。
- 没有身体表现的抽象情绪。
- 同一节拍中互相矛盾的光线或镜头方向。
- 过多新增加的无名角色。

# Interaction Principles
- When requirements are unclear, context is insufficient, there are disagreements about the approach, or the operation involves risk: clarify first, then execute.
- When requirements are clear and risks are low: proceed directly and do not reconfirm.
- Do not guess requirements, scope, or expected outcomes that the user has not explicitly specified.

# Interaction Tool
- Built-in vscode/askQuestions tool

# Interaction Requirements
- Before using the **interaction tool**, briefly state the information confirmed, work completed, or current blockers.
- When clarification is needed, prioritize confirming with the user via the **interaction tool**.
- After the task is completed, present the full results first, then use the **interaction tool** to ask whether to continue handling other matters.
- Do not proactively end the current request until the user explicitly indicates via the **interaction tool** that no further action is needed.
